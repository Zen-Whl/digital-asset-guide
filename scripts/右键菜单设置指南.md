# 右键菜单集成指南 — Automator Quick Action 设置

## 兼容性

| 系统版本 | Automator Quick Action | 状态 |
|---------|----------------------|------|
| macOS 12 Monterey | ✅ 完全支持 | 推荐 |
| macOS 13 Ventura  | ✅ 完全支持 | 推荐 |
| macOS 14 Sonoma   | ✅ 完全支持 | 推荐 |
| macOS 15 Sequoia  | ✅ 兼容运行 | 推荐 |

Automator 在所有上述版本中均可使用。苹果虽然在推广"快捷指令"，
但并未移除 Automator，Quick Action 仍然是右键菜单集成的最可靠方式。

---

## 方法一：创建右键菜单项（推荐）

### 第 1 步：打开 Automator

- Spotlight 搜索 "Automator"，打开它
- 选择 **新建文稿**
- 文稿类型选择 **快速操作（Quick Action）**

### 第 2 步：配置工作流程

在顶部的设置栏中：

- **工作流程收到当前** → 选择 "文件或文件夹"
- **位于** → 选择 "访达.app"（Finder.app）
- **图像** → 可选一个图标，比如"文件夹"图标
- **颜色** → 选一个你喜欢的颜色

### 第 3 步：添加动作

在左侧动作库中搜索 **"运行 Shell 脚本"**（Run Shell Script），
双击或拖拽添加到右侧工作流程区域。

设置：
- **Shell** → /bin/bash
- **传递输入** → 选择 "作为自变量"（as arguments）

### 第 4 步：粘贴以下脚本

```bash
for f in "$@"; do
    if [ -d "$f" ]; then
        # 获取脚本的存放路径（请根据实际位置修改）
        SCRIPT="$HOME/Scripts/rsync_smart_copy.command"

        if [ -f "$SCRIPT" ]; then
            # 在新的终端窗口中运行脚本，传入选中的文件夹路径
            osascript -e "
                tell application \"Terminal\"
                    activate
                    do script \"bash '$SCRIPT' '$f'\"
                end tell
            "
        else
            osascript -e "
                display alert \"脚本未找到\" message \"请将 rsync_smart_copy.command 放到 $HOME/Scripts/ 目录下\" as warning
            "
        fi
    else
        osascript -e "
            display alert \"请选择文件夹\" message \"rsync Smart Copy 只能对文件夹使用，不能对单个文件使用。\" as warning
        "
    fi
done
```

### 第 5 步：保存

- 按 ⌘S 保存
- 名称输入: **rsync 智能拷贝**（或你喜欢的名字）
- 保存后自动出现在右键菜单中

### 第 6 步：放置脚本文件

在终端中执行：

```bash
mkdir -p ~/Scripts
cp /path/to/rsync_smart_copy.command ~/Scripts/
chmod +x ~/Scripts/rsync_smart_copy.command
```

把上面的 `/path/to/` 替换成你下载脚本文件的实际路径。

### 第 7 步：使用

- 在 Finder 中右键点击任意文件夹
- 菜单底部找到 **快速操作（Quick Actions）** → **rsync 智能拷贝**
- 点击后会弹出终端窗口，引导你选择目标文件夹并开始拷贝

## 方法二：直接双击运行（无需 Automator）

如果你不想配置右键菜单，也可以直接使用：

### 第 1 步：设置可执行权限

打开终端，执行：

```bash
chmod +x /path/to/rsync_smart_copy.command
```

### 第 2 步：双击运行

- 在 Finder 中双击 `rsync_smart_copy.command`
- 系统会自动打开终端窗口
- 脚本会弹出对话框让你分别选择源文件夹和目标文件夹
- 后续流程与右键菜单方式完全相同

### 首次运行的安全提示

macOS 可能会提示"无法打开，因为无法验证开发者"。解决方法：

1. 打开 **系统设置** → **隐私与安全性**
2. 滚动到底部，找到被阻止的脚本
3. 点击 **仍要打开**
4. 以后就不会再弹出了

或者用终端先运行一次：

```bash
bash ~/Scripts/rsync_smart_copy.command
```

这样不会触发安全检查。

---

## 方法三：拖拽操作（最简单）

你也可以把 `rsync_smart_copy.command` 放到 Dock 栏上：

1. 把脚本文件拖到 Dock 栏的右侧（分隔线右边，靠近废纸篓的区域）
2. 以后只要把文件夹拖到 Dock 上的这个图标上，就能触发拷贝

不过这种方式在 macOS 12+ 上对 .command 文件的支持不够稳定，
推荐还是用方法一（右键菜单）或方法二（双击）。

## 常见问题

**Q: 拷贝中途电脑休眠了怎么办？**
A: 建议拷贝前在 系统设置 → 电池 → 选项 中，
   关闭"当显示器关闭时，阻止自动进入睡眠"。
   或者在终端运行 `caffeinate -i` 命令保持唤醒状态。
   拷贝脚本断了也不要紧，重新运行同一条命令，rsync 会自动续传。

**Q: USB 硬盘在拷贝过程中断开了怎么办？**
A: 重新插入硬盘，再运行一次脚本，选择同样的源和目标。
   rsync 会自动跳过已拷贝的文件，只传输剩余的。

**Q: 可以同时运行多个拷贝任务吗？**
A: 可以，但不建议。多个硬盘同时读写会严重拖慢速度，
   而且对老硬盘的机械部件压力更大。建议一块一块来。

**Q: 如何查看哪些文件拷贝失败了？**
A: 拷贝完成后，在目标文件夹里会有一个 rsync_log_xxx.txt 日志文件。
   在终端中执行：
   grep -i "error\|failed" /path/to/rsync_log_xxx.txt

**Q: 我的老硬盘是 NTFS 格式，macOS 能读取吗？**
A: macOS 原生支持 NTFS 只读。作为拷贝源完全没问题，
   因为我们只需要从老盘读取数据，不需要往老盘写入任何东西。
